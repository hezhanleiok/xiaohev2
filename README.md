# xiaohev2

一个简洁、美观、完全独立的客户端界面。本项目让用户能够**手动添加和管理自己的服务器配置**，所有数据均安全地存储在用户本地浏览器中，**无需任何在线订阅或外部依赖**。

这个项目是学习如何构建一个现代 PWA (Progressive Web App)、使用 `localStorage` 进行持久化状态管理以及实现纯客户端应用的绝佳示例。

## ✨ 功能特性

-   **完全独立与隐私**: 无需在电脑上运行任何辅助软件，不依赖任何在线订阅服务。所有服务器数据都只存在于您自己的浏览器中。
-   **多种导入方式**: 支持手动输入、从剪贴板导入、从订阅URL导入以及通过摄像头扫描二维码添加节点或订阅。
-   **智能国家识别**: 添加服务器时，应用会自动根据服务器IP识别其所在国家，并显示对应的国旗图标。
-   **本地持久化**: 您的所有配置都会安全地保存在浏览器的 `localStorage` 中，关闭或刷新页面后数据依然存在。
-   **自动分组**: 应用会根据您为服务器设置的“分组”名称，自动将服务器列表进行归类和展示，一目了然。
-   **节点管理**: 清晰地展示所有服务器节点，并支持按延迟、名称等多种方式排序。
-   **主题切换**: 内置设置中心，支持浅色（Light）和深色（Dark）主题一键切换。
-   **PWA 支持**: 支持添加到主屏幕，提供接近原生应用的离线体验。

## ⚙️ 工作原理

本应用是一个纯粹的前端应用，其核心工作流如下：
1.  **用户输入**: 用户通过“添加服务器”弹窗以多种方式（手动、剪贴板、URL、扫码）导入服务器配置。
2.  **本地存储**: 应用接收到输入后，将该服务器配置对象保存到浏览器的 `localStorage` 中。
3.  **读取与展示**: 每次应用启动时，它会从 `localStorage` 读取所有已保存的服务器配置，并将其渲染到界面上，按分组进行展示。

**用户 <==> 浏览器界面 (本项目) <==> localStorage (浏览器本地数据库)**

> **注意**: 当前版本仅完成了服务器的本地管理和 UI 交互。由于浏览器的安全限制，Web 端无法直接建立底层网络连接。要实现真正的连接功能，需要将此项目打包成原生应用（见“未来展望”）。

## 🚀 如何使用

1.  **部署或本地运行**:
    *   **推荐**: 将此项目文件夹部署到任何静态网站托管服务上（如 Vercel, Netlify, GitHub Pages）。
    *   **本地**: 在现代浏览器中直接打开 `index.html` 文件。
2.  **添加您的第一个服务器**:
    *   应用首次打开时，会显示一个欢迎界面。
    *   点击右上角的 **“+”** 图标。
    *   在“添加服务器”弹窗中，选择您喜欢的方式（如“扫码”或“剪贴板导入”）。
    *   完成后，返回主界面，您刚刚添加的服务器就会出现在列表中了！
